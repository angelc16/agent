🎯 Qué buscan

Un demo funcional de un agente conversacional (ej. en Telegram o chat simple) que permita al usuario, mediante lenguaje natural, crear una campaña y dentro de ella crear un evento.
El sistema, al crear el evento, debe disparar la integración con WhatsApp (Whapi) para que se genere un grupo, y luego devolverle al usuario la URL de ese grupo.

📌 Reglas y criterios clave

1. Interacción natural: el usuario no usa comandos técnicos, el bot debe guiar con preguntas (ej. nombre campaña, fecha evento, etc.).

2. Flujo mínimo:

Crear campaña (usar Okolo como company y Lukia Whapi DEV como integration → datos hardcoded).

Crear evento (fecha futura obligatoria, imagen puede ser dummy, admins con número de cel).

El evento dispara creación de grupo(s) de WhatsApp vía Whapi (tiene ~1 min de delay).

Capturar el link generado y devolvérselo al usuario.

Campos:

Contexto: opcional pero recomendable llenarlo con descripción del evento.

Metadata: ignorar.

Limitaciones:

Delay en WhatsApp para crear grupos (el bot debe manejar respuesta pendiente → “aún en proceso, intenta luego”).

Los leads no interactúan con la plataforma, solo se capturan vía este flujo.

📂 Entregable

Demo funcional donde se vea el flujo de conversación → creación de campaña → creación de evento → entrega de URL del grupo.

Validación: que en el admin dev de Lukia aparezcan creadas la campaña y el evento.
